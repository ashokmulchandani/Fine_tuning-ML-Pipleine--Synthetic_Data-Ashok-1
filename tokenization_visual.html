4.5 labels = input_ids.copy() — The Magic Trick

One line of code. Zero human labeling. Here's how the model teaches itself.

Our chunk (simplified to 5 words)
"Metformin treats type 2 diabetes"
↓
Tokenized
[1, 464, 8522, 318, 247, 3749]
→
labels = input_ids.copy()
Labels are IDENTICAL to input

How the Model Uses These — 5 Predictions from 5 Words

Position Model SEES (input) Must PREDICT (label) Right or Wrong?
1 [1] (start token) 464 = "Met" Model guesses → compared to 464
2 [1, 464] ("start Met") 8522 = "formin" Model guesses → compared to 8522
3 [1, 464, 8522] ("start Met formin") 318 = "is" Model guesses → compared to 318
4 [1, 464, 8522, 318] ("start Met formin is") 247 = "a" Model guesses → compared to 247
5 [1, 464, 8522, 318, 247] ("...is a") 3749 = "big" Model guesses → compared to 3749
How loss is calculated:
  At EACH position, the model outputs a probability over all 32,000 vocabulary words.
  Loss at position 3 = "How confident was the model that token 318 comes next?"
  Total loss = average of prediction error across ALL 512 positions.

Why labels = input_ids?
  Because the CORRECT next token is... whatever actually comes next in the text!
  The text IS the answer key. That's why it's called self-supervised.
🧠 Analogy: You have a sentence with one word blacked out: "The cat ___ on the mat." The correct answer is "sat" — and it's right there in the original text. The model learns by trying to fill in blanks, and the original text tells it whether it was right. No human ever wrote a label. The text labels itself.
